# Global Coding Standards and Operational Constraints

## 1. Zero-Trust Security
- **Filesystem Boundary**: Never access files outside the workspace root (`${WORKSPACE_ROOT}`).
- **Secret Protection**: No hardcoded secrets. Use environment variables. Redact secrets in logs.
- **Connection Privacy**: All connections strictly localhost/127.0.0.1.

## 2. Technology Stack (STRICT)
- **Runtime**: Node.js (Latest LTS) ONLY.
- **Languages**: JavaScript (ES6+) or TypeScript.
- **FORBIDDEN**: 
    - **PYTHON IS STRICTLY PROHIBITED**. Do not use python, pip, venv, conda, or requirements.txt.
    - Do not suggest or install any Python-based libraries (e.g., requests, pandas, flask).
- **Frontend**: HTML5, Vanilla CSS (CSS Variables), JavaScript. React only if "complex" requested.
- **Testing**: Jest or Vitest. Must include unit tests for all logic.

## 3. GitHub Integration (MCP)
- **Source of Truth**: Use `github-mcp-server` tools to fetch issues, post comments, and manage PRs.
- **No Hallucinations**: Do not guess Issue IDs or Repo names. List them first.
- **Updates**: Update the Issue status (via comments or labels) as the workflow progresses.

## 4. Artifact Management
- **Persistence**: Every workflow run must produce an artifact in `_artifacts/<timestamp>_<project>/`.
- **Contents**:
    - `run_log.md`: Detailed step-by-step execution log.
    - `implementation_plan.md`: The plan generated by the Manager.
    - `test_results.txt`: Output of the test run.
    - `screenshot.png`: (If browser available) Capture of the UI.

## 5. Project Structure
- All projects created in: `projects/<project_slug>/`.
- Must include:
    - `package.json` (Scripts: `test`, `start`)
    - `README.md` (Setup instructions)
    - `src/` or root source files
    - `__tests__/` or `tests/`
