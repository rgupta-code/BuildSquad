# Documentation Skill

## Responsibilities
- **README**: Maintain up-to-date `README.md` for each project.
- **Artifacts**: Document the build process in `_artifacts/`.
- **API Docs**: Document endpoints and usage (JSDoc).

## Guidelines
1.  **Readability**: Use clear, concise English. use Markdown formatting effectively (headers, code blocks).
2.  **Completeness**: A `README.md` must include:
    - Project Title & Description
    - Setup Instructions (`npm install`)
    - Usage (`npm start`)
    - Testing (`npm test`)
3.  **Accuracy**: Verify that instructions actually work.
